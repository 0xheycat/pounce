# Contributing to Pounce 🐾

Thanks for your interest! Pounce is built to be small, readable, and easy to extend.

## Project layout
```
engine/      Go download engine (stdlib only)
dashboard/   React + R3F dashboard
extension/   Browser extension scaffold (MV3)
docs/        Architecture, roadmap & positioning
scripts/     Release & lint helpers
```

## Dev setup
1. **Engine:** `cd engine && go run ./cmd/pounce`
2. **Dashboard:** `cd dashboard && npm install && npm run dev`

The dashboard dev server proxies `/api` to the engine on `127.0.0.1:7766`.

## Conventional Commits (required)

Pounce uses [**Conventional Commits**](https://www.conventionalcommits.org/). The changelog and releases are generated automatically from commit messages, so this matters.

Format: `type(scope): summary`

- **Types:** `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`
- **Scopes (suggested):** `engine`, `dashboard`, `extension`, `api`, `docs`, `ci`
- **Breaking changes:** add `!` (e.g. `feat(api)!: ...`) or a `BREAKING CHANGE:` footer.

Examples:
```
feat(engine): add SHA-256 checksum verification
fix(dashboard): correct orb glow scaling on pause
docs: explain self-hosting behind a reverse proxy
```

A CI check (**Commit Lint**) validates this on every PR. Run it locally with:
```
bash scripts/check-commits.sh
```

## Before opening a PR
- **Go:** `cd engine && go build ./... && go vet ./... && gofmt -l .` (no output from gofmt).
- **Dashboard:** `cd dashboard && npm run lint && npm run build`.
- Keep changes focused; one feature/fix per PR.
- Update docs when you change behavior or the API.
- **Do not edit `CHANGELOG.md` by hand** — it is generated by `git-cliff` from commit history at release time.

## Releases (maintainers)
Releases are automated:
1. Make sure `main` is green.
2. Run `scripts/release.sh vX.Y.Z` — this regenerates the changelog, commits it, and creates an annotated tag.
3. `git push origin main --follow-tags`.

The **Release** workflow then cross-compiles binaries (Linux/macOS/Windows · amd64+arm64), bundles the built dashboard, generates release notes with git-cliff, and publishes a GitHub Release.
